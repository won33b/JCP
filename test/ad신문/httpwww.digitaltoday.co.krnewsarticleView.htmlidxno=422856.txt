		<h4 class="subheading">오류 여부 사전에 발견하기 위한 가상 테스트베드<br />
지역에서 발생한 오류, 전국으로 확산하는 것을 차단할 수 있는 시스템도 부재 <br />
과기정통부, '네트워크 안정성 확보방안' 마련할 계획</h4>		
		<!-- contents //-->
		<article  id="article-view-content-div" class="article-veiw-body view-page" itemprop="articleBody">
		
			<div style="text-align:center">
<figure class="photo-layout image photo_412403 float-center" data-idxno="412403" data-type="photo" style="display:inline-block; max-width:600px"><img alt="KT 네트워크 구조 [이미지 : 과기정통부]" src="https://cdn.digitaltoday.co.kr/news/photo/202110/422856_412403_1348.jpg" />
<figcaption>KT 네트워크 구조 [이미지 : 과기정통부]</figcaption>
</figure>
</div>

<p>[디지털투데이 백연식 기자] 과학기술정보통신부는 KT 네트워크 장애 사고에 대한 원인에 대해 KT가 전국을 하나로 연결한 대규모 네트워크를 안전장치&nbsp;없이 운영했으며, 오류를 사전에 차단할 장치도 마련하지 않고 있는 등 KT의 부실한 관리 체계 때문이라고 29일 밝혔다. 또 네트워크가 차단된 가상 상태에서 오류 여부를 사전에 발견하기 위한 가상 테스트베드가 없었고, 지역에서 발생한 오류가 전국으로 확산하는 것을 차단할 수 있는 시스템도 부재했다는 것이 과기정통부의 설명이다.</p>

<p>과기정통부에 따르면 지난 25일 오전 11시16분경부터 KT DNS(Domain Name System&middot;도메인 주소를 IP 주소로 변환하는 역할을 수행) 서버에서 트래픽이 급증했다. 중앙 1차 DNS(혜화)의 경우 평시 대비 22배 이상, 중앙 2차 DNS(혜화)는 평시 대비 4배 이상, 부산 DNS는 평시 대비 3.7배 이상 증가하는 등 큰 폭의 트래픽 증가가 나타났다.</p>

<p>과기정통부는 이번 사고 로그기록을 분석한 결과, 부산국사에서 기업 망 라우터 교체 작업 중 작업자가 잘못된 설정 명령을 입력해 사고가 발생했다고 설명했다. 이 라우팅 오류로 인해 전국적인 인터넷 네트워크 장애가 발생했다는 설명이다.</p>

<p>해당 작업자의 작업내역을 확인한 결과, 사고발생 라우터에 라우팅 설정명령어 입력과정에서 IS-IS(내부 라우터 간 경로정보를 주고받는 프로토콜, 통상 1만개 이하의 경로정보를 교환) 명령어를 마무리하는 부분에서 &lsquo;exit&rsquo; 명령어를 누락한 것으로 확인됐다. 이로 인해 BGP(외부 라우터와 경로정보를 주고받는 프로토콜, 통상 수십만개 수준의 경로정보를 교환)에서 교환해야 할 경로정보가 IS-IS로 전송됐다는 것이다.</p>

<p>통상 1만개 내외의 정보를 교환하는 IS-IS 프로토콜에 수십만개의 BGP 프로토콜의 정보가 잘못 전송되면서, 라우팅 경로에 오류가 발생하게 됐다는 설명이다.</p>

<p>과기정통부 조사 결과, KT 네트워크 내에 있는 라우터들을 연결하는 IS-IS 프로토콜은 잘못된 데이터 전달에 대한 안전장치 없이 전국을 모두 하나로 연결하고 있었다. 결국 한 개 라우터의 잘못된 라우팅 경로 업데이트가 전국의 라우터에 연쇄적으로 일어나서 장애가 전국적으로 확대됐다는 설명이다.</p>

<p>또 KT의 IPTV 서비스와 음성전화서비스에 장애가 발생한 이유로는 서비스 망은 별도로 구성됐으나, 인터넷 서비스 장애로 인해 문자 이용이 늘고 단말 전원을 리셋하는 이용자들이 늘어나면서 트래픽 증가가 발생하여 부하가 가중됐을 것으로 추정했다.</p>

<p>조사반은 디도스 공격 여부에 대한 조사 결과, DNS 서버에 대한 트래픽 증가는 있었지만, 시스템 자원 디도스 공격 및 네트워크 대역폭 공격은 확인되지 않았다고 설명했다.</p>

<p>과기정통부는 당초 라우터 교체 작업이 야간에 예정됐으나, 주간에 수행된 이유 역시 KT의 관리적&middot;기술적 문제 때문이라고 파악했다. 작업 관리자 없이 KT 협력업체 직원들인 작업자들끼리만 라우팅 작업을 수행하는 등 작업오류를 방지하기 위한 작업관리체계가 부실했으며, 네트워크가 연결된 채로 작업이 이뤄졌다는 것이다.</p>

<p>1&middot;2차에 걸친 사전검증 단계가 존재했으나, 사람이 직접 검토하는 체계이기 때문에 오류를 발견하지 못했다. 또 네트워크가 차단된 가상 상태에서 오류 여부를 사전에 발견하기 위한 가상 테스트베드가 없었고, 지역에서 발생한 오류가 전국으로 확산하는 것을 차단할 수 있는 시스템도 부재했다는 것이 과기정통부의 설명이다.</p>

<p>과기정통부는 이번 조사결과를 바탕으로, 주요통신사업자 네트워크의 생존성&middot;기술적&middot;구조적인 대책이 담긴 &lsquo;네트워크 안정성 확보방안&rsquo;을 마련할 계획이다. 과기정통부 네트워크정책실장을 단장으로 네트워크 전문가 등 관계 전문가들과 TF를 구성해 운영 및 추진한다. 네트워크 안정성 확보방안은 단기대책과 중장기대책을 포괄하는 방안이 검토될 예정이다.</p>

<p>단기 대책으로는 주요통신사업자의 네트워크 작업체계&middot;기술적 오류확산 방지체계 등 네트워크 관리체계를 점검하고, 주요통신사업자가 네트워크 작업으로 인한 오류여부를 사전에 진단할 수 있는 시뮬레이션 시스템을 도입한다.</p>

<p>또 주요통신사업자가 승인된 작업계획서의 내용 및 절차가 준수되는지에 대해 네트워크관제센터에서 기술적 점검 체계를 구축토록 하고, 라우팅 설정오류로 인한 피해를 최소화하기 위해 주요통신사업자가 라우팅 작업을 할 때 한 번에 업데이트되는 경로정보 개수를 일정 수준 이하로 제한 등이 검토될 계획이다.</p>

<p>중장기 대책으로는 주요통신사업자의 통신장애 대응 모니터링 체계 강화, 네트워크 안정성과 복원력을 높이는 기술개발, 안정적인 망 구조 등 네트워크의 생존성 확보를 위한 구조적 대책 마련 등도 추진할 예정이다.</p>

<p>KT는 이용자 피해현황 조사 및 피해구제 방안 마련을 추진하고, 방송통신위원회는 이용자 피해구제 방안 이행여부를 점검할 예정이다. 방송통신위원회는 또 통신장애 발생시 실효성 있는 피해구제를 위한 법령 및 이용약관 등 개선방안 마련을 검토할 계획이다.</p>		</article>

					if((fixWingBannertop + nonScrollingHeight_L) < winScrollTop) {

						bannerAllHeight = articleTitleHeight + scrollingHeight_L + 1;								// 상단 타이틀 + 스크롤배너의 총 높이
						siteAllHeight = winHeight - userFooterHeight + downMargin;									// 윈도우 높이 + 사이트 Footer 높이

						if( (winScrollTop >= ((docHeight - userFooterHeight) - winHeight)) && (bannerAllHeight >= siteAllHeight) ) {					// 사이트 Footer 넘어가는 경우 처리
							wingBnTop_L -= ((userFooterHeight + downMargin) - (winHeight - (scrollingHeight_L + topMargin + articleTitleHeight)));
							$('#wingBanner .wing-banner-left .wing-banner.scroll-on').animate({top:wingBnTop_L+"px" },{queue: false, duration: duration});		// 마지막 배너 움직임 처리
						}
					} else {
						wingBnTop_L = 0;																			// 최상단 처리
					}
					$_this.animate({top:wingBnTop_L+"px" },{queue: false, duration: duration});

				} else {																							// 우측 날개

					var wingBnTop_R = winScrollTop - fixWingBannertop + topMargin - nonScrollingHeight_R;			// 날개배너 스크롤 위치 (가변)

					if( articleTitleHeight ) wingBnTop_R += articleTitleHeight;										// 기사본문 시 상단 스크롤 타이틀이 나타나면 스크롤 타이틀 만큼 날개배너 위치 조정

					if((fixWingBannertop + nonScrollingHeight_R) < winScrollTop) {

						bannerAllHeight = articleTitleHeight + scrollingHeight_R + 1;								// 상단 타이틀 + 스크롤배너의 총 높이
						siteAllHeight = winHeight - userFooterHeight + downMargin;									// 윈도우 높이 + 사이트 Footer 높이

						if( (winScrollTop >= ((docHeight - userFooterHeight) - winHeight)) && (bannerAllHeight >= siteAllHeight) ) {					// 사이트 Footer 넘어가는 경우 처리
							wingBnTop_R -= ((userFooterHeight + downMargin) - (winHeight - (scrollingHeight_R + topMargin + articleTitleHeight)));
							$('#wingBanner .wing-banner-right .wing-banner.scroll-on').animate({top:wingBnTop_R+"px" },{queue: false, duration: duration});		// 마지막 배너 움직임 처리
						}
					} else {
						wingBnTop_R = 0;																			// 최상단 처리
					}
					$_this.animate({top:wingBnTop_R+"px" },{queue: false, duration: duration});
				}
			}
		});
	}

});
